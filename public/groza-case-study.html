The Problem

A Los Angeles-based learning center needed their WordPress/Elementor website brought into ADA/WCAG 2.1 AA compliance, GDPR cookie consent implemented, and tracking scripts properly managed. The site had 100+ accessibility errors across 30+ pages, 200+ color contrast failures, tracking scripts firing without consent, and an accessibility overlay widget (UserWay) that increased legal liability rather than reducing it.

UserWay Overlay Removed

Disabled the UserWay accessibility overlay widget. Reason: overlay widgets increase legal liability (active class action lawsuit against UserWay Feb 2026, 1,023 companies sued in 2024 while using overlays, National Federation of the Blind formally opposes overlays). Replaced with native, code-level ADA fixes.

Technical Approach

All fixes applied via WordPress Code Snippets plugin (server-side PHP). Zero theme files modified. Zero plugin files modified. Every change is fully reversible by deactivating the snippet.

Key PHP Techniques Used

// Alt text injection via WP_HTML_Tag_Processor (WordPress 6.2+)
$processor = new WP_HTML_Tag_Processor($content);
while ($processor->next_tag('img')) {
  if (!$processor->get_attribute('alt')) {
    $processor->set_attribute('alt', $derived_alt);
  }
}
// Color contrast fix via str_ireplace (site-wide CSS override)
add_filter('style_loader_tag', function($html) {
  // Inject contrast-corrected CSS after each stylesheet loads
  $fixes = '.btn { color: #5a6a08 !important; }';
  return $html . '<style>' . $fixes . '</style>';
});
// GDPR script blocking via PHP output buffer
ob_start(function($html) {
  // 1. Strip hardcoded tracking scripts with preg_replace
  // 2. Store scripts in JS function in page footer
  // 3. Read cookieyes-consent cookie
  // 4. Load scripts only after "advertisement" consent
  // 5. Listen for real-time consent changes (no page refresh needed)
  return $modified_html;
});
